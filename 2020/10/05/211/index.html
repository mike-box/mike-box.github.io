5534. 可见点的最大数目

给你一个点数组 points 和一个表示角度的整数 angle ，你的位置是 location，其中 location = [posx, posy] 且 points[i] = [xi, yi]都表示 X-Y 平面上的整数坐标。

最开始，你面向东方进行观测。你 不能 进行移动改变位置，但可以通过 自转 调整观测角度。换句话说，posx 和 posy不能改变。你的视野范围的角度用angle 表示， 这决定了你观测任意方向时可以多宽。设 d 为逆时针旋转的度数，那么你的视野就是角度范围[d - angle/2, d + angle/2] 所指示的那片区域。

对于每个点，如果由该点、你的位置以及从你的位置直接向东的方向形成的角度 位于你的视野中 ，那么你就可以看到它。

同一个坐标上可以有多个点。你所在的位置也可能存在一些点，但不管你的怎么旋转，总是可以看到这些点。同时，点不会阻碍你看到其他点。

返回你能看到的点的最大数目。

思路

  1. 题目还是非常有意思的题目，刚开始看到貌似无法着手，实际就是一个数学问题。
  2. 我们可以将问题转化为以location为圆心，在弧度的大小固定时，该弧度内所包含的点数最多是多少。题目中给定的实际弧度为$\frac{angel*2\pi}{180}$.
  3. 首先我们求出所有的点以location为原点的弧度，在此我们为了方便可以全部转化为角度来计算。我们可以利用正切函数，快速的求出角度。然后将所有点的角度按照从小到大进行排序。
  4. 我们遍历题目中每一个角度，然后利用二分查找或者双指针，探测在角度范围内[arr[i],arr[i] + angel]有多少个元素。
  5. 此题稍微trickly的一点在于角度的处理问题，因为arr[i] + angel]有可能超过$180^{。}$，此时我们就需要处理，第一种直接将其映射到$[-180^{。},0^{。}]$;另一种我们直接在后面在后面扩展$[180^{。},540^{。}]$.另外对于与location相同的点，我们需要将其筛选出来单独处理。

题意

小学奥赛题目

解题思路

  1. 其实比赛的时候觉得这题挺奇怪的，后来仔细思考一下，确实如题解中所描述的，跟九连环的游戏特别像。如果要改变第n位的状态，必须要将第n-1位置为1,且低于n-1位的全部置为0.
    九连环要想下/上第n个环，就必须满足两个条件(第一个环除外)。一、第n-1个环在架上；二、第n-1个环前面的环全部不在架上。玩九连环就是要努力满足上面的两个条件。
  2. 我们可以看到解答，首先知道九连环可以用gray码来解决，那是纯数学的方式，另一种方法我们可以用递归dfs解决，找到规律即可。
  3. 实际情况中我们只需要两种基本的操作即可实现所有的数的变换：
  • 把当前数xxxx变为全零0000即可;
  • 把当前数xxxx的首位变为1,其余为0,即1000即可;
    推论如下：
  • lower(n): 表示将n变为0的最小操作次数。
    如果当前数字的二进制形式为1xxxxx,则我们只需要将xxxxx全部变为10000，此时数变为110000，此时我们再将110000变为10000，这样再重复10000变为0操作即可完成,我们定义该操作为lower(n).
  • upper(n): 表示将n变为2^x的最小操作次数。
    如果当前数字的二进制形式为1xxxxx,则我们只需要将xxxxx全部变为0，此时数变为100000; 如果当前数字的二进制形式为0xxxxx,我们首先需要将xxxxx变为10000,然后将最高为变为1，然后将除最高位意外剩余的位变为0. 按照上述的推论，我们可以得到以下递推公式：
