5198. 丑数 III

题目

请你帮忙设计一个程序，用来找出第 n 个丑数。

丑数是可以被 a 或 b 或 c 整除的 正整数。

示例 4：

地址

https://leetcode-cn.com/contest/weekly-contest-155/problems/ugly-number-iii/

题意

求满足题目要求的最小整数。

思路

  1. 刚开始对这道题目没有思路，后来看了答案才知道解法，原理为荣斥定理；
  2. 我们可以通过计算小于等N的数中有多少个满足题目要求的丑数，然后采用二分查找法即可找到最终的目的。
  3. 我们可以假设目前只有两个a，b，能被a或b整除，给定N，求小于等N的数中有多少个数能够满足被a或者b整除，数学公式如下
    $F[N] = N/a + N/b - N/((ab)/gcd(a,b))$,从而我们可以推导出3个数的情况；
    $F[N] = N/a + N/b + N/c - N/((a    b)/gcd(a,b)) - N/((ac)/gcd(a,c)) - N/((bc)/gcd(b,c)) + N/((abc)/gcd(a,b,c))$
  4. 我们通过查找满足刚好只有n个数的整数X，通过二分查找即可找到目标结果。

5199. 交换字符串中的元素

题目

给你一个字符串 s，以及该字符串中的一些「索引对」数组 pairs，其中 pairs[i] = [a, b] 表示字符串中的两个索引（编号从 0 开始）。

你可以 任意多次交换 在 pairs 中任意一对索引处的字符。

返回在经过若干次交换后，s 可以变成的按字典序最小的字符串。

示例 1:


示例 2：

示例 3：

地址

https://leetcode-cn.com/contest/weekly-contest-155/problems/smallest-string-with-swaps/

题意

求交换后字符的字典序的最小值。

思路

  1. 我们知道定理，在一个可以互相交换的连通集合内，通过无数次的交换，肯定能够满足排序字典最小的状态.
  2. 我们分别找到所有的可交换的连通区域，然后再对每个区域内所有的字符进行排序，按照字典序最小的状态进行排序.
  3. 将排序后的字符再重新组合成最新的字符串即可得到结果，但该题比较恶心的时，容易超时，所以处理时要注意压缩.
