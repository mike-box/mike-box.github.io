5147. 递减元素使数组呈锯齿状

题目

给你一个整数数组 nums，每次 操作 会从中选择一个元素并 将该元素的值减少 1。

如果符合下列情况之一，则数组 A 就是 锯齿数组：

每个偶数索引对应的元素都大于相邻的元素，即 A[0] > A[1] < A[2] > A[3] < A[4] > …
或者，每个奇数索引对应的元素都大于相邻的元素，即 A[0] < A[1] > A[2] < A[3] > A[4] < …
返回将数组 nums 转换为锯齿数组所需的最小操作次数。

示例 2：

https://leetcode-cn.com/contest/weekly-contest-148/problems/decrease-elements-to-make-array-zigzag/

题意

找到数组变为波浪形数组的最小操作次数。

思路

  1. 题目中的波浪数组即为数组中的奇数位比相邻的偶数位的数字小，或者数组中的所有偶数位的数字比相邻的奇数位的数字小；
  2. 我们只需要求出这两种变动方式下，数组的数需要变动的最小值；
  3. 只需要求出，如果使得数组中的对应位的数变为比相邻的数都小的最小变动的值；

5148. 二叉树着色游戏

题目

有两位极客玩家参与了一场「二叉树着色」的游戏。游戏中，给出二叉树的根节点 root，树上总共有 n 个节点，且 n 为奇数，其中每个节点上的值从 1 到 n 各不相同。

游戏从「一号」玩家开始（「一号」玩家为红色，「二号」玩家为蓝色），最开始时，

「一号」玩家从 [1, n] 中取一个值 x（1 <= x <= n）；

「二号」玩家也从 [1, n] 中取一个值 y（1 <= y <= n）且 y != x。

「一号」玩家给值为 x 的节点染上红色，而「二号」玩家给值为 y 的节点染上蓝色。

之后两位玩家轮流进行操作，每一回合，玩家选择一个他之前涂好颜色的节点，将所选节点一个 未着色 的邻节点（即左右子节点、或父节点）进行染色。

如果当前玩家无法找到这样的节点来染色时，他的回合就会被跳过。

若两个玩家都没有可以染色的节点时，游戏结束。着色节点最多的那位玩家获得胜利 ✌️。

现在，假设你是「二号」玩家，根据所给出的输入，假如存在一个 y 值可以确保你赢得这场游戏，则返回 true；若无法获胜，就请返回 false。

示例：

地址

https://leetcode-cn.com/contest/weekly-contest-148/problems/binary-tree-coloring-game/

题意

求在第一个选手选择着色后，第二个选手选择着色能否赢。

思路

  1. 刚开始看到这个题目，没有理解题目的意思，后来仔细分析题意发现该题一点都不难，但是需要仔细理解题意。
  2. 题目中的关键信息点位于：两位玩家轮流进行操作，每一回合，玩家选择一个他之前涂好颜色的节点，将所选节点一个 未着色 的邻节点（即左右子节点、或父节点）进行染色。
  3. 仔细分析，我们得出实际上玩家二在玩家一选择的节点的左孩子，右孩子或者父节点中这三个节点中选择一个即为玩家二所能占领的最大节点数。可以仔细分析下题意就会发现。
  4. 因此我们只需要求出节点x的左子树的个数或者右子树的个数，或者节点x的为根构成的子树以外的节点的数目，取其中的最大值，如果有一个值能够满足大于n/2,则我们认为玩家二会赢。

5149. 快照数组

题目

实现支持下列接口的「快照数组」- SnapshotArray：

  • SnapshotArray(int length) - 初始化一个与指定长度相等的 类数组 的数据结构。初始时，每个元素都等于 0。
  • void set(index, val) - 会将指定索引 index 处的元素设置为 val。
  • int snap() - 获取该数组的快照，并返回快照的编号 snap_id（快照号是调用 snap() 的总次数减去 1）。
  • int get(index, snap_id) - 根据指定的 snap_id 选择快照，并返回该快照指定索引 index 的值。

示例：

地址

https://leetcode-cn.com/contest/weekly-contest-148/problems/snapshot-array/

题意

主要是求快照的值，每次快照时就会保存之前快照的值。

思路

  1. 仔细分析一下，实际上不需要我们每次进行快照时将数组中所有的值，进行保存一次，我们只需要要保存哪些值被修改过的快照即可。
  2. 实际上我们可以用数组或者map来保存当前索引，每次快照的值，本题为了简单，我们用hashmap保存快照值。
  3. set操作时时，我们只需要将索引对应的快照赋给当前的val即可。
  4. snap操作时，我们只需要将snapid值加一
  5. get操作时，我们只需要找到该索引小于等于当前snapid时的第一个值。
